# Ideas

Random thoughts collected here. No particular order.

## Tools

A small CLI that watches a directory and prints a summary of what changed
since the last time you ran it. Like `git status` but for arbitrary
directories, without needing a repo.

## Writing

Start keeping a proper log of what I read. Not summaries, just titles and
a sentence about why I picked it up.

## Workflow

Try blocking the first hour of the morning for deep work before opening
any messaging apps. The context-switch cost of checking messages first
thing is real.

## Project ideas

- A minimal static site generator that compiles Markdown to HTML with
  zero configuration and sane defaults
- A tool that converts meeting notes to action items automatically
